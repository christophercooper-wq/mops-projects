Issues

1. Lead Acceleration

Problem: Database Events / ETL Process During Application

  • Current State: 40-45 minute lag between application signup and HubSpot visibility
  • Root Cause: Batch warehouse syncs instead of real-time event streaming
  • Business Impact: Sales cannot act on high-intent leads in real-time ("speed-to-lead" vs "lead acceleration" distinction)
  • Evidence: See 01-root-cause-analysis.md for detailed user journey analysis

Sub-Issues:

  • Signup Services only emits events on final form submission (not intermediate steps)
  • No visibility into application progress (step 1/7 completed, step 3/7 abandoned, etc.)
  • Marketing Ops receives "app submitted" but Sales needs "app stuck at step 5" alerts
  • No progress state cache (Redis/Firebase) for real-time dashboard

2. Workflow Efficiency

Problem: Existing Sprawl, User/Data Flow

  • Current State: 300+ active HubSpot workflows, many conflicting or redundant
  • Root Cause: No governance, no documentation, no sunset policy, multiple people creating workflows over 3+ years
  • Business Impact: 15% of contacts have sync errors due to workflow conflicts

Specific Conflicts Identified:

  1. MQL Status Assignment: 3 workflows set lifecyclestage = "MQL" with conflicting logic
  2. Event Follow-Up: 40 individual event workflows (should be 1 template)
  3. Attribution Updates: 47 workflows reference "Primary Inbound Channel" (3 different properties)
  4. Email Triggers: Overlapping enrollment criteria causing duplicate emails

Symptoms:

  • Workflows executing in wrong order (Workflow B overwrites Workflow A's changes)
  • Contacts stuck in "pending" states (waiting for workflow that was deactivated)
  • Property values changing unexpectedly (multiple workflows updating same property)

3. Field Maintenance

Problem: Data Schema, Taxonomy, & Sync Properties

  • Current State: No canonical schemas between systems, same concept represented 4 different ways
  • Root Cause: Each system (Signup Services, Warehouse, Salesforce, HubSpot) uses different field names/formats
  • Business Impact: 15% sync error rate HubSpot ↔ Salesforce

Examples:

Concept Signup Services Warehouse Salesforce HubSpot Issue
User Progress application_step (int 1-7) app_progress_pct (float 0-100) Application_Status__c (picklist) app_status_v2 (text) No ETL contract, breaks on changes
Company Size employee_count (int) company_size (text range) NumberOfEmployees (int) company_employee_count (number) Type mismatches cause sync failures
Attribution utm_campaign (text) first_touch_campaign (text) Lead_Source_Campaign__c (text) 3 different properties (legacy, L3, native) Conflicting data, 47 workflows broken

Specific Property Issues:

  • Tier 5 (Legacy/Unknown): ~200 properties with 0% fill rate, no documentation, unknown purpose
  • Orphaned Mappings: HubSpot properties that sync to deleted Salesforce fields (causes errors)
  • Type Mismatches: Text → Picklist, Date format differences (MM/DD/YYYY vs YYYY-MM-DD)

4. Lifecycle Campaigns (Demand Generation)

Problem: Poorly Defined Audience Goals, Target Operator Discrepancies

  • Current State: Role defined as "Lifecycle Marketing Manager" but Leadership wants Demand Generation
  • Root Cause: CRO (Tommy McNulty) focused on TOF/pipeline, JD asked for MOF/BOF expertise
  • Business Impact: 3 qualified candidates (7/7 scores) rejected due to strategic misalignment

Organizational Questions (Unanswered):

  • What team owns the data infrastructure? (Eng? Data? RevOps? Marketing Ops?)
  • What team owns tool acquisition / experimentation? (Growth? Marketing?)
  • How does the org support MOF/BOF initiatives? (No budget, no ownership)
  • Who defines lifecycle stages? (RevOps? Marketing Ops? Sales?)

Role Clarity Needed:

  • Growth: Rapid customer acquisition, short-term testing, TOF/MID-funnel optimization, paid acquisition, SEO, CRO
    • Metrics: CAC, MQL/SQL volume, conversion rates, acquisition velocity
  • Lifecycle: Long-term retention, customer journey orchestration, post-purchase activation, retention, expansion, winback
    • Metrics: LTV, churn rate, retention, engagement, expansion revenue

Current Gap: Growth team has zero direct reports, Christopher (Marketing Ops) reports to Anthony (RevOps) not Jeremy (Growth)

5. Database Cleanup

Problem: Tiering & Value Per Contact / LTV

  • Current State: Contact database includes 3+ years of dead contacts, skewing metrics
  • Root Cause: No sunset policy, no contact scoring, no archival process
  • Business Impact: Inaccurate property fill rates (24% fill rate includes 70% dead contacts → actual 8% for live contacts)

Specific Issues:

  • Property Fill Rate Bias: "Primary Inbound Channel" shows 24% filled, but:
    • Dead contacts (3+ years old): 90% of database, 8% fill rate
    • Active contacts (< 6 months): 10% of database, 85% fill rate
    • Reporting: "Only 24% of contacts have attribution" → Misleading, live contacts have 85%
  • HubSpot Cost Bloat: Paying per contact, but 70% are dead (never converted, 3+ years inactive)
  • List Segmentation: Dynamic lists include dead contacts, inflating counts

Need:

  • Contact tiering (Tier 1: Customers, Tier 2: Active leads, Tier 3: Nurture, Tier 4: Dead)
  • LTV scoring (prioritize high-value accounts)
  • Archival process (move Tier 4 to separate database, remove from HubSpot)

Potential Solutions

Intermediate Goals (Skydog Deliverables - 2-3 Months)

1. Cleaner, More Reliable Data

Skydog Scope:

  • Audit all existing data flows and automation rules (Phase 1)
  • Implement data validation and normalization standards (Phase 2)
  • Eliminate redundant workflows, properties, and legacy fields (Phase 2)

Approach:

1. Workflow Audit (Phase 1, Weeks 1-2):

  • Pull all workflows via API, export to Airtable
  • For each workflow, extract:
    • Trigger criteria (which properties)
    • Enrollment criteria (which lists/properties)
    • Actions (which properties set/updated)
    • Last modification date, last enrollment date
    • Total enrolled (all-time and last 90 days)
  • Output: Dependency matrix showing which properties are used where

2. Property Schema Rationalization (Phase 1, Weeks 2-3):

  • Categorize all properties:
    • Tier 1: Core (name, email, company) - Keep
    • Tier 2: Attribution (source, campaign) - Keep but standardize
    • Tier 3: Lifecycle (MQL date, SQL date) - Rationalize
    • Tier 4: Integration (SFDC ID, Snowflake ID) - Audit sync
    • Tier 5: Legacy/Unknown - Archive candidates
  • For each property, document: 
    Property: application_status_signup
Used in Workflows: [ID 123, ID 456, ID 789]
Used in Forms: [Contact Form, Event Registration]
Fill Rate: 12% (of non-dead contacts)
Last Modified: 18 months ago
Created By: [Unknown]
Purpose: [Undocumented]
Recommendation: [Archive - deprecated by app_status_v2]

3. Workflow Consolidation (Phase 2, Weeks 4-5):

  • Consolidate 40 event workflows → 1 template workflow (uses dynamic properties)
  • Merge 3 MQL workflows → 1 master workflow (clear criteria)
  • Deactivate orphaned workflows (0 enrollments in 90 days: ~50 workflows)
  • Deactivate duplicates ("Copy of...", "Test...": ~20 workflows)
  • Target: 300+ active workflows → <100 workflows

DRI:

  • Christopher (Marketing Ops): HubSpot workflow audit, property rationalization
  • Skydog Ops: Audit tool build, analysis, recommendations

2. Faster Speed to Lead

Skydog Scope:

  • Reduce lag between lead signup and HubSpot visibility (Phase 2)
  • Improve sync latency between HubSpot, Salesforce, and self-serve signup funnel (Phase 2)
  • Establish full visibility into what data comes in, when, and through which system (Phase 2)

Approach:

1. Current State Analysis (Phase 1, Week 2):

  • Map data flow: Signup Services → Warehouse → HubSpot → Salesforce
  • Measure latency at each step (webhook delays, batch sync frequencies, API rate limits)
  • Identify bottlenecks (is 40-minute lag due to warehouse batch? Or HubSpot sync?)
  • Document existing Salesforce APEX automation (Wexpert internal analysis reference)

2. Salesforce Sync Optimization (Phase 2, Weeks 5-6):

  • Field Mapping Fixes:
    • Text → Picklist mismatches (e.g., HubSpot industry text → SFDC Industry picklist)
    • Missing required fields (e.g., SFDC requires Company, 30% of HubSpot contacts missing)
    • Date format conversions (MM/DD/YYYY → YYYY-MM-DD)
  • Duplicate Detection:
    • Enable Salesforce duplicate rules (block duplicates based on email)
    • Add HubSpot dedupe logic before sync (check if SFDC ID exists, search for email match)
  • Sync Monitoring Dashboard:
    • Real-time visibility (Airtable table: Date, Object, Status, Error Message)
    • Slack alerts (if error rate >10% in 1-hour window)
    • Weekly report (automated email: last 7 days sync stats, error rate trend)
  • Target: 15% error rate → <5% error rate

3. HubSpot Sync Configuration Review (Phase 2, Week 6):

  • Which objects sync? (Contacts only? Or Contacts + Companies + Deals?)
  • Sync frequency? (Real-time vs. batch?)
  • Duplicate detection rules?
  • Identify configuration issues causing errors

DRI:

  • Anthony (RevOps): Salesforce field mapping, duplicate rules, APEX automation review
  • Skydog Ops: Sync analysis, optimization recommendations, monitoring dashboard build
  • Christopher (Marketing Ops): HubSpot configuration, property mapping

Note: Reducing 40-minute lag to <5 minutes requires Engineering involvement (event emission changes in Signup Services). This is out of scope for Skydog, but Skydog can document requirements and recommend architecture (see Phase 3 Advisory).
